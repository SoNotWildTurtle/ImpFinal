imp/core/imp-3d-neural-network.py - 3D neural network with fitness-tracked evolution, logging, and network summary metrics.
  Supports advanced neuron types via `spawn_advanced_neuron` and tracks usage counts.
  Includes `spawn_novel_neuron` to create uniquely typed neurons automatically.
  `spawn_novel_neuron_for_task` attaches a novel neuron to the most active node for a chosen task.
  `auto_evolve` reinforces busy connections and prunes weak links before logging results.
  `add_backup_connection` creates defensive myelin-rich shortcuts and
  `maintain_backup_connections` adjusts them during evolution.
  `guide_novel_neuron` links new neurons to active pathways so they can quickly adopt a role.
  `heal_from_ledger` requests blockchain-backed restoration without external models.
  `restore_repository_from_ledger` rebuilds the full codebase from the latest ledger snapshot during major attacks.
  `auto_restore_from_ledger` verifies the ledger and performs a full repository restore when many mismatches are found.
  `dynamic_restore_from_ledger` runs a verification helper and restores the entire repository when mismatches exceed a threshold or the chain is broken.
  `auto_evolve` logs average fitness and dormant neuron totals to `imp-evolution-log.json`.
  `evolution_trend` reads the last two evolution entries to highlight changes in
    neuron count, fitness, and other metrics.
  `evolution_trend_history` reports deltas across several recent entries for
    broader historical context.
  `reactivate_dormant_neurons` revives low-activity neurons when usage rises and `auto_evolve` records how many returned.
  Reawakened neurons reinforce their existing connections to stabilize revived pathways, and `auto_evolve` logs the reinforcement count.
  Connections include a task label so Schwann or oligodendrocyte neurons can
  reinforce task-specific pathways without degrading others.
  The `evolve` method marks low-usage neurons dormant rather than deleting them
  so the network can reawaken connections when needed.
    `auto_evolve` reinforces the busiest pathways, spawns a novel neuron automatically, and logs summaries to `logs/imp-evolution-log.json`.
    `auto_evolve` saves a snapshot to `logs/imp-network-snapshots/` before evolving.
    `simulate_evolution` returns an evolved copy for sandbox experiments.
    `visualization_data` + `export_visualization` produce node/edge payloads for
    3D dashboards, storing the latest export in `logs/imp-3d-visualization.json`.
    `spatial_summary` computes bounding boxes, centroid, spacing, and volume for
    richer 3D overlays.
    `task_regions` reports per-task clusters with dominant neuron types and
    distance from the network core.
    `novel_neuron_experiment_designs` produces experiment briefs for every
    experimental neuron, and `log_novel_neuron_experiments` archives them in
    `logs/imp-novel-neuron-experiments.json`.
    `novel_neuron_research_notes` summarises advanced neurons and
    `log_novel_neuron_research` archives recommendations in
    `logs/imp-novel-neuron-research.json`.
    `forward_by_angle` allows routing signals along connections that match a given
  direction in 3D space, separating tasks to reduce poisoning risk.
  `find_optimal_path` searches for the lowest-resistance route between two
  neurons using connection distance and myelin.
